静态方法static 是建立在类的基础上
与对象无关
如果static静态方法和变量设为public，他没被封装，不安全
可以用以下方法进行封装

// flower小区
private static String community = "flower";

// getInstance()方法——静态不用this，因为this指的是创建的对象，而静态属性与对象无关，只与上层的类有关
public static String getCommunityInstance(){
    return community;
}
// main()里调用
String str = Dogs.getCommunityInstance();

封装，继承，多态

封装：鼓励将数据和方法捆绑在一起形成新的单元，通过将属性设置为 public，失去了对数据的封装，外部代码可以随意访问和修改属性，这破坏了封装性。

继承extends
子类重写父类的方法 final写在方法前则不能子类不能重写（但是能用父类方法）
public final void xxx（）
类则不能被继承
值则无法被修改private static final String xxx（本来在当前类中是可以修改的）

多态
多个行为，功能多了
要有继承关系

// 替父从军 向上转型
HuaHu huaHu = new HuaMuLan();
System.out.println(huaHu.name);
System.out.println(huaHu.age);
huaHu.fight();

// 有一天，仗打完了，遇上自己心爱的人,做回自己 向下转型
HuaMuLan huaMuLan = (HuaMuLan)huaHu;
System.out.println(huaMuLan.name);
System.out.println(huaMuLan.age);
huaMuLan.dressing();
//OUTPUT
HuaHu
45
干架！
HuaMuLan
19
化妆！
